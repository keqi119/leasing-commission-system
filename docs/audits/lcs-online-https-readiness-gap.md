# LCS 线上 HTTPS 多角色目标差距审计

审计日期：2026-07-04
审计基线：main@27c7b90
目标：线上通过 HTTPS 实现多角色系统操作

## 1. 总体结论

当前架构距离线上 HTTPS 多角色系统还有较大差距。当前更接近“本地 MVP 原型 + SQLite 单机试用脚手架”，不具备生产线上系统所需的认证、会话、生产数据库、文件存储、审计、安全和部署能力。

线上目标不建议在补齐线下单机 V1 前启动。原因是当前真实数据录入、审核、试算、审批和导出页面闭环尚未打通，过早部署会把原型问题暴露给线上用户。

## 2. 能力差距表

| 能力项 | 当前状态 | 缺口 | 风险 | 建议阶段 |
| --- | --- | --- | --- | --- |
| 真实登录认证 | 缺失 | 当前 API 用 `x-lcs-role` 请求头模拟角色 | 任意人可伪造角色请求 | P1/P2 |
| 用户会话 | 缺失 | 无 cookie/session/token 生命周期 | 页面无法识别真实登录用户 | P1/P2 |

| HTTPS 部署 | 缺失 | 当前是本地 dev server | 不能安全公网访问 | P3 |
| 生产数据库 | 缺失 | 当前 SQLite + sql.js raw client | 并发、锁、事务、备份能力不足 | P3 |

| 日志审计 | 部分 | 有审批日志模型，但页面和 API 覆盖不足 | 操作追责不完整 | P2/P3 |

| 上传文件存储 | 缺失/部分 | 模板下载有，真实文件上传 main 未接 | 凭证、导入文件无法持久管理 | P2/P3 |
| 导出文件存储 | 部分 | ExportRecord 有 fileName/fileUrl 字段，API 返回 workbook | 缺对象存储/权限 URL/下载历史文件 | P2/P3 |

| 访问控制 | 部分 | API permission helper 存在 | 无路由保护、页面按钮保护、数据行级过滤 | P2/P3 |
| 数据隔离 | 缺失/部分 | 单部门/本地 fixture 为主 | 多部门、销售本人数据、押金敏感隔离不足 | P2/P3 |
| 公网安全 | 缺失 | 无 CSRF、速率限制、安全 header、上传扫描 | 暴露公网风险高 | P3/P4 |
| 文件凭证安全 | 缺失 | proof URL 主要为字符串 | 凭证访问控制和脱敏不足 | P3 |

| 性能容量 | 未验证 | 当前数据量小，测试为 fixture | 大账期导入/导出可能超时 | P4 |
| 数据质量治理 | 部分 | 导入校验服务存在 | 页面未打通，线上导入需更强校验和回滚 | P2/P3 |

## 3. 认证授权差距

当前认证实现：

- `apps/web/src/server/auth.ts:18-20` 从请求头读取 `x-lcs-role` 和 `x-lcs-user-id`。
- `packages/shared/src/auth.ts:36` 定义角色权限矩阵。
- 没有登录页、密码校验、session、cookie、token、退出登录、密码重置。

线上必须补齐：

1. 登录页和服务端认证。
2. session/cookie 或 token 机制。
3. 密码哈希/企业账号/SSO 之一。
4. 页面路由保护。
5. API 权限保护。
6. 行级数据权限：销售本人数据、部门数据、押金敏感数据。
7. 审计登录、登出、失败登录、越权访问。

建议：线下 V1 可先做本地账号登录；线上前再接企业账号或 SSO。

## 4. 数据库差距

当前状态：

- Prisma schema 覆盖核心模型。
- `packages/database/src/index.ts` 同时暴露 PrismaClient 和 sql.js raw client。
- 实际 workflow 大量使用 raw SQL。
- seed 默认写 `packages/database/prisma/dev.db`。
- H07 health 默认检查 `local-data/db/dev.db`。

线上必须补齐：

1. 统一开发、测试、本地试用、生产数据库路径/连接策略。
2. 从 sql.js raw client 迁移到正式 Prisma Client 或生产数据库驱动。
3. 引入 PostgreSQL/MySQL 等生产数据库。
4. 建立 migration 流程和回滚策略。
5. 明确事务边界：导入 commit、试算 snapshot、审批、导出绑定必须原子化。
6. 保证金额字段始终用 cents 整数。
7. 保留审批、导出、结算 run 的不可变历史。

## 5. 部署差距

当前状态：

- 本地 Next dev server。
- Windows 单机脚本和本地试用包。
- 无云部署配置、无 HTTPS、无生产环境变量管理。

线上必须补齐：

1. 选择部署平台和运行模式。
2. 配置 HTTPS 域名和证书。
3. 配置生产数据库连接。
4. 配置文件上传/导出存储。
5. 配置环境变量、密钥、日志。
6. 建立 CI/CD，至少包含 test/typecheck/build/prisma validate。
7. 建立回滚流程和数据库备份恢复流程。

## 6. 安全差距

线上前必须考虑：

- 登录暴力破解防护。
- CSRF 防护。
- 上传文件类型、大小、病毒或恶意内容控制。
- 导出文件权限控制和过期链接。
- 敏感字段脱敏：客户、车牌、凭证、账户信息。
- API 速率限制。
- 服务端错误不泄露内部路径。
- 审计日志不可篡改。
- 管理员操作二次确认。
- 备份加密与恢复演练。

## 7. 建议路线图

| 阶段 | 目标 | 必做能力 |
| --- | --- | --- |
| P1：线下单机可用 | 业务人员在一台 Windows 电脑完整跑账期 | 真实录入/导入、台账 CRUD、审核、试算、审批、导出、备份恢复、本地登录 |
| P2：局域网/内网试用 | 小团队多角色并发试用 | 会话、行级权限、局域网访问、DB 路径统一、基础审计、文件存储目录权限 |
| P3：线上 HTTPS | 公网或企业内网 HTTPS 多角色系统 | 生产数据库、正式认证、HTTPS、对象存储、CI/CD、备份、权限审计、安全基线 |
| P4：生产增强 | 稳定运营 | 监控告警、性能优化、复杂报表、自动对账、企业账号深度集成 |

## 8. 线上前置门槛

不建议在以下事项完成前部署线上 HTTPS：

1. 线下 V1 完整闭环通过。
2. 所有核心页面具备真实 DB 操作能力。
3. 不再依赖 `x-lcs-role` 作为最终权限来源。
4. 导入、审批、结算、导出都有不可变审计记录。
5. DB 路径和数据库访问层统一。
6. 导入文件和导出文件有权限控制和存储策略。
