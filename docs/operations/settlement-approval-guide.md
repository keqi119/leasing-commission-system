# 结算审批闭环操作指南

## 老板如何驳回

进入 `/commission/approvals`，查看 HR 提交的结算 run、风险提示和差异说明。

驳回时必须填写原因，例如：

```text
财务补审 2026-04-21 租金收入后重算。
```

系统会把当前 run 标记为 `REJECTED`，并保留历史快照。HR 不能基于 rejected run 导出正式发放表。

## HR 如何重算

老板驳回后，HR 等待相关角色修正数据或人工调整审批完成，然后重新发起试算。

重算会生成新的 `runNo`，例如：

```text
2026-04-RUN-001 -> REJECTED
2026-04-RUN-002 -> SUBMITTED / APPROVED
```

新 run 不覆盖旧 run。

## 财务修正数据后如何重新进入试算

财务补审租金收入或外调利润后，HR 重新计算。系统会生成新 run，并在差异页展示：

- 部门实收变化
- 达成率变化
- 提成池变化
- 每名销售贡献收入和应发金额变化

## 审批通过后为什么不能直接修改

老板审批通过后，该 run 成为正式版本。周期进入锁定状态，收入、外调利润、押金、指标和人工调整不能直接修改。

需要修正时必须走重开流程：

```text
申请重开 -> 老板 / 管理员审批 -> 保留旧 approved run -> 重新试算 -> 新 run 审批通过后成为最新正式结果
```

## 导出结果为什么必须绑定 approved run

导出表是工资发放依据，必须绑定一个已经审批通过的 runNo。这样后续即使周期重开，历史导出记录仍能追溯到当时的正式审批版本。
