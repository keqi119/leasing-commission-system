# LCS-P1-H04 验收清单

## 审批闭环

- [ ] HR 可以生成 run v1。
- [ ] HR 可以提交 run v1 给老板审批。
- [ ] 老板可以驳回 run v1 并填写原因。
- [ ] rejected run 不能导出正式奖金表。
- [ ] HR 修正后可以生成 run v2。
- [ ] run v2 不覆盖 run v1。
- [ ] 系统能展示 run v2 与 run v1 的差异。
- [ ] 老板审批通过 run v2 后，HR 只能基于 approved run 导出。

## 人工调整

- [ ] 人工调整必须填写原因。
- [ ] 未审批人工调整不进入提成。
- [ ] 审批通过人工调整进入新 run。
- [ ] 人工调整不修改原始收入台账。
- [ ] 导出表显示调整金额。

## 锁定与重开

- [ ] 审批通过后周期锁定。
- [ ] 锁定周期禁止导入影响结算的数据。
- [ ] 重开必须填写原因。
- [ ] 重开后保留原 approved run。
- [ ] 旧导出记录仍绑定原 runNo。

## 试运行报告

- [ ] 可以创建试运行问题。
- [ ] 可以解决试运行问题。
- [ ] 可以完成试运行并生成报告。
- [ ] 报告引用正确的审批 runNo。
- [ ] 报告包含发现问题、已解决问题和遗留风险。

## 回归验证

- [ ] H02 验收场景仍通过。
- [ ] H03 导入模板、预览、提交、导出金标准仍通过。
- [ ] `pnpm check:project-isolation` 通过。
- [ ] `pnpm seed:acceptance` 通过。
- [ ] `pnpm test` 通过。
- [ ] `pnpm typecheck` 通过。
- [ ] `pnpm prisma:validate` 通过。
- [ ] `pnpm build` 通过。
