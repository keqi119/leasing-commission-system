# 暂缓功能清单

| 功能 | 优先级 | 触发条件 | 本阶段不做原因 | 未来建议阶段 |
| --- | --- | --- | --- | --- |
| 部门应收超过阈值后整体提成池打折 | P1 | 连续试运行后确认应收阈值和折扣规则 | H02 只验证未收款不参与提成，折扣口径仍需业务确认 | LCS-P2 |
| 银行流水自动对账 | P1 | 财务月结数据量明显增加 | 需要银行接口或导入模板，超出 MVP 验收加固范围 | LCS-P2 |
| 邮件自动识别月度指标 | P2 | 指标确认邮件格式稳定 | 本阶段已有手工目标录入和审批流程 | LCS-P3 |
| Excel 历史数据导入 | P1 | 试运行需迁移历史订单和回款 | 需要清洗模板和失败回滚策略 | LCS-P2 |
| 押金风险看板 | P2 | 押金争议记录达到固定复盘频率 | H02 只做结算页风险提示和台账记录 | LCS-P3 |
| 多部门多规则集 | P2 | 第二个部门开始使用系统 | MVP 先固定租赁销售部口径 | LCS-P2 |
| 多法人、多账套 | P3 | 财务主体拆分或跨法人结算 | 当前团队规模不需要复杂账套 | LCS-P4 |
| 工资系统对接 | P2 | HR 确认工资发放系统接口 | H02 只导出正式发放表，不做自动发薪 | LCS-P3 |
| 操作审计高级筛选 | P3 | 审批日志和操作日志量明显增加 | 当前保留日志数据结构和基础记录即可 | LCS-P3 |
| 正式登录鉴权接入企业账号 | P1 | 试运行用户超过本地角色头模式 | H02 重点是业务验收，不引入企业身份系统复杂度 | LCS-P2 |
